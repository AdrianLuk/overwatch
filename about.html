About Electron

Electron is an open source framework developed by GitHub. It allows for the development of cross platform desktop applications by using Node.js to handle the backend processes and Chromium (a web browser developed by Google) for the front-end. Electron is the framework behind several popular desktop applications such as the Atom text editor, the Discord chat service, and the team collaboration tool Slack.

Electron enables developers to create native desktop applications with web technologies such as HTML, CSS, and JavaScript. Thus, developers only need to learn these few languages instead of other programming languages such as Python, Java, or C++. Building an application in electron is almost identical to building a website because you only need HTML, CSS, and Javascript

To get started, you will need to have Node and npm (node package manager) installed. After installing them, you will need to create a package.json file. This file is extremely important as it contains all the details about the application itself. Fortunately, thanks to npm init, npm makes it easy for developers to set up their package.json file. Electron will then take a main JavaScript file (usually named main.js) and execute it. This file creates the windows of the application that contain the rendered web pages that you create.

When starting up an application using Electron, what is created is called the main process. This process is responsible for interacting with your operating system and creates the windows of the application. The main process uses the BrowserWindow module to create the application windows and each browser window then runs its own renderer process. This renderer process takes the web page that has been written in HTML, CSS, and JavaScript and displays it in the window. You can see that whenever a new browser window is created, it will show up as a new process in the task manager. All renderer processes are created with the main process.

The main process can access the native GUI using modules available directly in Electron. While the native GUI (Graphical User Interface) of the operating system is usually only accessible through the main process, there are techniques that allow for some of that work to be done in the renderer processes. For example, you can typically only access the developer tools from the main process and not from the JavaScript file for your renderer processes, but with the help of something called Inter-Process Communication (or IPC for short), you can create a button that acts as a remote so that when it is pressed, it can open the developer tools as if it was from the main process. Click the Open/Close Dev Tools button to see this IPC in action.
